Cheesy name, I know. I was on the spot when uploading the app to the server and I had to find a name which wasn't already taken. Maybe I'll change it at some point.
My motivation for building this project is to learn to build an application from end to end using Python, Flask, HTML, CSS, JavaScript, PostgreSQL, SQLAlchemy, object-relational mapping, model-view-controller architecture patterns and other concepts. I'm trying to be pretty thorough in documenting the process; check out the release log if you're curious. Feel free to email me if you have questions or would like the source code. I'm a big believer that helping others learn benefits everyone.
The goal of this project is to make a realistic stock trading game from scratch. The current feature set is pretty bare-bones, but here's where we're going...

StockHawk trading game concept:

  • Upon sign-up, users receive $1 million of play money which can be used to trade stocks within the game.
  • All trades are based on actual market conditions. We track how much you've gained or lost on a particular transaction, pay dividends when applicable, etc.
  • Game leaderboards give you bragging rights for wealthiest, most active and other categories.

Here are some features which I'm considering developing:

  • Smart search (not just stock symbols, but also company names.)
  • Ability to trade on margin and short sell.
  • Realistic capital gains/income tax liability implications for transactions.
  • Stock graphs/charts (most likely using JavaScript.)
  • Historical stock data lookup.
  • Stats on the most active/best/worst-performing stocks of the day.
  • Links to articles you might be interested in, based on the stocks you look at or own.
  • Resources on how to learn more about the stock market and trading.
  • Email notifications.
  • Sign in with either username or email address and password.
  • Possibility to sign in or create an account via Facebook or other social media.
  • Account settings/password management.
  • Private messaging.
  • "Follower" relationships and other social interactions.
  • Some decent design, including a logo.
  • User avatars.
  • Let me know if you have other ideas!
In the meantime, check out the stock page.
